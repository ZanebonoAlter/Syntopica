## Context

当前 `BoardDailyReportTimeline.vue` 使用扁平卡片列表展示日报，所有内容块（今日重点、板块动态、叙事线索）用相同的卡片样式，视觉层次缺失。TagsPage 整体布局限宽 1440px，匹配详情面板不跟随滚动，文章时间筛选缺少快捷操作。

本次变更聚焦前端 UI 重构，不涉及后端 API 或数据模型变更。所有数据复用现有 `GET /api/semantic-boards/:id/daily-reports` 和 `GET /api/daily-reports/:id` 端点。

## Goals / Non-Goals

**Goals:**
- 日报 tab 从平铺卡片改为双态展示：概要列表 + 全屏旧报纸弹窗
- 旧报纸弹窗使用泛黄纸张底色 + 微弱噪点 + 衬线字体的报纸排版风格
- 按章节分页：第 1 页 = 今日重点 + 板块动态，后续每页 = 一个叙事线索章节
- 左右边缘按钮翻内容页，顶栏上下按钮切换日报日期
- TagsPage 默认 tab 改为「文章」
- 布局宽度放宽至 1800px，匹配详情面板 sticky
- 文章时间筛选增加快捷 chip（今天/3天/7天/30天）
- 克制派动画：功能性转场，不炫技

**Non-Goals:**
- 不改动后端 API、数据模型、日报生成流程
- 不改动日报数据结构（highlights/dynamics/sections/threads）
- 不引入新的 npm 依赖
- 不改动其他页面（首页、文章详情等）
- 不做首字下沉等重度排版装饰

## Decisions

### D1: 日报双态展示 — 概要列表 → 全屏旧报纸弹窗

**选择**：默认收起态（概要列表）+ 点击进入全屏弹窗（旧报纸风格），替代现有平铺展开。

**替代方案**：
- A）保持展开/收起，优化卡片内部排版 → 视觉层次改善有限，交互路径不变
- B）PPT 式一页一天，无概要列表 → 需要先翻到某天才能看内容，缺少全局概览
- C）双列报刊式 → 数据结构不支持高度压缩的摘要展示

**理由**：双态设计兼顾了快速浏览（概要列表一眼扫多天）和深度阅读（全屏报纸弹窗）。弹窗遮罩建立"进入阅读空间"的心理暗示，旧报纸风格与信息消费场景契合。

### D2: 旧报纸排版 — 泛黄纸张 + 衬线字体

**选择**：弹窗内容区使用 `#f4eed7` 泛黄纸张底色 + CSS 微弱噪点纹理 + 边缘泛黄渐变。标题使用衬线字体（Noto Serif SC），正文保持无衬线字体。

**理由**：泛黄纸张 + 衬线字体的组合建立"阅读旧报"的视觉心智模型，在暗色 App 内形成强烈对比，突出阅读空间。中度做旧（不做折痕、不做不规则边缘）保持阅读舒适度，不影响信息获取。

### D3: 全屏弹窗结构 — 遮罩 + 居中纸张面板

**选择**：点击概要卡片后弹出全屏遮罩（rgba(0,0,0,0.7)）+ 居中纸张面板。面板结构：顶栏（日期 + 上下切换 + 关闭按钮）、纸张内容区（按章节分页）、左右边缘翻页按钮。

**替代方案**：
- A）组件内 inline 展开 → 缺乏空间隔离感，容易被其他内容干扰
- B）独立路由页面 → 过重，离开上下文，返回体验差
- C）侧边抽屉 → 空间不够展示报纸排版

**理由**：全屏弹窗在视觉上建立独立的阅读空间，不离开 TagsPage 上下文。遮罩点击可关闭，降低认知负担。

### D4: 按章节分页（方案 B）

**选择**：日报内容按章节分页 — 第 1 页 = 今日重点 + 板块动态，后续每页 = 一个叙事线索 section。每页显示当前页码 `n / N`。

**替代方案**：
- A）固定高度分页 → 内容可能在段落中间截断，阅读体验差
- B）全部内容一页滚动 → 内容长时翻不到头，失去"报纸翻页"体验

**理由**：按章节分页保证内容不截断，每页有明确主题。典型日报（2-3 个 highlights + dynamics + 2-4 个 sections）= 3-5 页，翻页次数适中。

### D5: 导航模型 — 左右翻页 / 上下换天

**选择**：
- **左右边缘按钮**：纸张两侧的半透明箭头按钮，翻内容页（同日内章节翻页）
- **顶栏上下按钮**：`[↑上一天] 5.27 星期二 [↓下一天]`，切换日报日期
- **键盘支持**：← → 翻页，↑ ↓ 换天，Esc 关闭弹窗

**理由**：两个维度的导航（内容页 vs 日报天）用不同方向区分，避免混淆。顶栏合并日期显示和换天操作，四个方向导航不互相干扰。

### D6: 默认 tab 改为「文章」

**选择**：TagsPage 选中 board 后默认激活「文章」Tab。

**理由**：用户进入板块最常做的是查看匹配文章（高频操作），板块内容配置是低频管理操作。默认 tab 应匹配最高频场景。

### D7: 布局宽度 min(1800px, 95vw)

**选择**：`.tags-main` 和 `.tags-topbar-inner` 的 max-width 从 1440px 改为 min(1800px, 95vw)。

**理由**：1440px 在 2K/4K 屏上留白过多。1800px 上限保持内容可读性，95vw 保证小屏不溢出。

### D8: MatchDetailPanel sticky 定位

**选择**：给 `.tags-match-detail-panel` 添加 `position: sticky; top: 1rem; align-self: flex-start;`，配合 `max-height: calc(100vh - 6rem); overflow-y: auto;`。

**理由**：最小改动（3 行 CSS），不改 DOM 结构。面板在文章列表滚动时保持可见，内容溢出时独立滚动。

### D9: 时间筛选快捷 chip

**选择**：在现有 filter chips 下方新增一行 [今天] [3天] [7天] [30天]，默认选中「今天」。

**理由**：最高频场景是看今天的文章，现在要手动选两次日期太重。快捷 chip 一键完成，保留 date input 作为 fallback。

### D10: 动画原则 — 克制派

**选择**：功能性动画，不炫技。只做弹窗出入、内容翻页和概要列表入场，不做视差、打字机效果、hover 微动效等装饰性动画。

**理由**：报纸阅读的核心是内容消费。动画服务于空间感和节奏感，而非视觉表演。

### D11: 弹窗出入动画

**选择**：弹窗打开 — 遮罩 fade-in 200ms + 纸张面板 scale(0.95→1) + fade-in 300ms。弹窗关闭 — 反向 200ms。概要列表 → 弹窗切换不做 card → page 的 morph 动画。

**理由**：scale + fade 是最克制的弹窗入场，建立"展开报纸"的空间感，不做过度拟物。

### D12: 概要列表 staggered fade-in

**选择**：概要卡片依次淡入，每张 `200ms duration`，间隔 `50ms delay`，同时 `translateY(8px → 0)`。Board 切换或从弹窗返回时触发。

**理由**：staggered entrance 创造阅读节奏，7 张卡片总时长约 500ms，舒适不拖沓。

### D13: 内容翻页动画

**选择**：左右翻页使用 `translateX` 水平滑动，`300ms ease-out`。右翻页（→）：当前页向左滑出，新页从右滑入。左翻页（←）：当前页向右滑出，新页从左滑入。使用 Vue `<Transition>` 实现。

**理由**：水平 slide 建立"翻报纸"的空间感，300ms 是功能性动画的舒适区间。

## Risks / Trade-offs

- **[弹窗首次加载延迟]** → 点击概要卡片时先显示 loading 骨架屏，日报详情数据量小（通常 < 10KB），延迟可接受
- **[概要列表信息密度低]** → 概要卡片只显示日期 + summary 截断前 30 字。但这是设计意图——快速浏览不需要完整信息，需要时点击进入报纸弹窗
- **[布局宽度变更影响]** → 1440px → 1800px 可能影响个别组件的固定宽度布局。需检查 sidebar 和内容区的 flex 布局是否自适应
- **[默认 tab 变更]** → 老用户习惯「板块内容」tab，突然变成「文章」可能需要适应。影响轻微，用户可通过 tab 切换
- **[按章节分页的边界情况]** → 如果日报只有 highlights 没有 sections，则只有 1 页。如果 sections 很多（>6），页数过多。但实际日报生成逻辑限制了 sections 数量，不构成实际风险
- **[Noto Serif SC 字体加载]** → 需要从 Google Fonts 或本地加载，首次加载可能有 FOUT。可通过 `font-display: swap` 缓解
