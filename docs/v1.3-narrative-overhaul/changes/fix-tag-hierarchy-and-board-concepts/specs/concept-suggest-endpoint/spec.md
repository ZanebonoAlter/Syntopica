## ADDED Requirements

### Requirement: LLM-driven concept suggestion endpoint
The system SHALL expose `POST /hierarchy/concepts/suggest` that accepts a `category` parameter (default "event") and returns a list of suggested board concept names and descriptions generated by LLM analysis of existing tag labels and descriptions.

#### Scenario: Suggest concepts for event category
- **WHEN** `POST /hierarchy/concepts/suggest` is called with `{"category": "event"}`
- **THEN** the system SHALL load active event tags without board concept assignments, extract their labels and descriptions, send them to LLM with a prompt asking for 3-5 concept suggestions, and return `{"success": true, "data": [{"name": "...", "description": "..."}, ...]}`

#### Scenario: Suggest concepts for empty category
- **WHEN** `POST /hierarchy/concepts/suggest` is called with `{"category": "person"}`
- **THEN** if fewer than 10 tags exist in that category, SHALL return `{"success": true, "data": []}` with no LLM call

#### Scenario: LLM call fails gracefully
- **WHEN** the LLM call for concept suggestion fails or returns unparseable JSON
- **THEN** the system SHALL return `{"success": false, "error": "..."}` and log the error without crashing

### Requirement: Suggestion does not create database records
The suggest endpoint SHALL only return suggestions as JSON. It SHALL NOT create `board_concepts` rows, generate embeddings, or modify any database state.

#### Scenario: Suggest is read-only
- **WHEN** `POST /hierarchy/concepts/suggest` is called
- **THEN** no rows in `board_concepts` SHALL be inserted, updated, or deleted

### Requirement: Suggestion prompt uses tag labels and descriptions only
The LLM prompt SHALL include only existing active tag labels and descriptions for the given category. It SHALL NOT include article content, embeddings, or clustering results.

#### Scenario: Prompt content
- **WHEN** building the LLM prompt for concept suggestion
- **THEN** the prompt SHALL contain: the category name, a list of up to 50 tag labels with their descriptions, and instructions to return a JSON array of concept suggestions with name (2-6 characters) and description (30-80 characters) fields
