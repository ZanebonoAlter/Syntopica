## 1. 手动创建板块

### 什么时候用

- 你很清楚自己想要追踪什么话题（比如"我就要一个 AI 板块"）
- 系统刚启动，还没有积累足够的小标签来触发升级建议

### 操作步骤

1. 点击左侧边栏底部的 **「+ 添加板块」** 按钮
2. 弹出对话框，填写基本信息：
   - **名称**（必填）：板块名字，比如「AI与机器学习」
   - **描述**（可选）：一句话说明这个板块覆盖什么，比如「人工智能、深度学习、大模型相关新闻」
   - **排序**：数字越小越靠前，默认 0
   - **受保护**：勾选后不会被自动删除
3. 点击 **「下一步」**，进入选择构成标签
4. 系统会根据你填的名称+描述，自动推荐一批相似的小标签。你可以：
   - 直接在列表中勾选需要的小标签
   - 在搜索框输入关键词过滤（比如搜"AI"只看 AI 相关的小标签）
   - 翻页查看更多候选
   - 一个都不选也可以（空构成标签的板块暂时不会匹配到任何文章）
5. 点击 **「确认创建」**

### 之后做什么

创建后板块会立即出现在左侧列表。但是 —— 历史文章不会自动归到这个新板块。你需要：

1. 点击左侧的 **「匹配回填」** 按钮，选择「全部标签」或「指定板块」模式
2. 等待回填完成（可以看到进度条）

## 2. 板块升级建议

### 这个功能是什么

系统读了很多文章后，积累了大量"小标签"（辅助标签）。当某个小标签出现了 5 次以上，系统就认为它够格升级。你点击按钮，系统会自动把相似的小标签分堆，然后问 LLM："这堆标签是不是可以立为一个板块？"

### 操作步骤

1. 点击左侧边栏的 **「✨ 升级建议」** 按钮
2. 弹窗打开，显示当前候选情况：
   - 「候选标签 X 个 · 聚类 Y 个」—— 候选数 = 出现了 5 次以上的小标签；聚类数 = 系统按语义把它们分成了几堆
   - 如果候选不够，会显示「暂无满足条件的升级候选」
3. 点击 **「获取 LLM 建议」** 按钮，系统开始分析
4. 等待几秒到十几秒（取决于候选数量和 LLM 响应速度）
5. 分析完成后，每条建议有三种可能的结果：

   | 建议类型 | 标记色 | 含义 |
   |---|---|---|
   | **创建新板块** | 🟢 绿色 | LLM 认为这堆标签够格成为一个独立的板块，给出了板块名称和描述 |
   | **合并到已有板块** | 🔵 蓝色 | 这堆标签跟某个已有板块高度重合，建议加进那个板块的构成里 |
   | **跳过** | ⚪ 灰色 | LLM 觉得这堆标签太分散，不值得成立板块 |

6. 每一条建议旁有 **「确认执行」** 按钮，点击后：
   - 创建新板块 → 立即生成板块 + 写入构成标签
   - 合并到已有 → 候选标签加入目标板块的构成
   - **面板不会关闭**，你可以继续处理下一条建议

7. 处理完后，面板顶部会显示提示：「已执行升级建议。历史标签归属不会自动回填，可手动触发匹配回填」

### 重新生成建议

如果你处理了部分建议、手动调整了板块构成、或候选池变化了，可以点击面板顶部的 **「重新生成建议」** 按钮重新分析。

### 小提示

- 升级建议是 LLM 给出的，不一定 100% 准确。你可以选择性确认，不满意的跳过
- 建议的名字和描述如果不太满意，确认创建后可以手动编辑板块
- LLM 升级不会自动回填。确认后需要手动触发回填，历史文章才会归入新板块

## 3. 管理板块构成标签

### 这是什么

每个板块由一组"小标签"（辅助标签）构成。当一篇文章的标签带了这些"小标签"时，文章就会归入这个板块。

比如「AI圈」板块的构成标签是 `[AI, OpenAI, 大语言模型, 深度学习]`。如果一篇文章的标签带了「AI」这个小标签，文章就归入 AI圈。

### 查看构成标签

选中左侧某个板块后，右侧上方会显示「构成标签」面板，列出所有构成小标签。每个小标签显示：
- **标签名**（如「AI」）
- **引用次数**（被多少篇文章引用过）
- **移除按钮**（鼠标悬停出现 ✕）

### 添加小标签到板块

1. 在构成标签面板右上角点击 **「添加」** 按钮
2. 面板展开一个选择器，系统自动推荐与板块最相似的小标签（按相似度从高到低排序）
3. 你也可以用搜索框筛选（比如搜「模型」找相关标签）
4. 勾选需要的小标签
5. 点击底部的 **「确认添加 N 个标签」** 按钮
6. 添加后显示提示：「已添加构成标签。历史标签归属不会自动回填，可手动触发 board 回填」

> ⚠️ 添加后需要手动触发「匹配回填」→「指定板块」模式，历史文章才会按新构成重新匹配。

### 从小标签移除

鼠标悬停在小标签上，点击 **✕** 按钮，确认后移除。

> ⚠️ 移除也不会自动回填。

## 4. 查看标签的板块归属

### 位置

- 在浏览文章列表时，每篇文章下方有标签行。每个标签右侧可能显示所属板块的标记
- 一个标签可以同时属于多个板块（最多 3 个），按匹配得分从高到低排列

### 点击标签

点击某个标签后，右侧会展示：
- 这个标签关联的所有小标签
- 这个标签归属的所有板块（及匹配原因）

## 5. 匹配回填

### 什么时候需要回填

| 场景 | 回填模式 |
|---|---|
| 你新建了一个板块，想让历史文章也归进去 | `全部标签` 或 `指定板块` |
| 你修改了板块的构成标签（添加/移除），想让改动生效 | `指定板块` |
| 你调整了匹配参数（比如降低了相似度门槛），想让更多文章归入 | `全部标签` |
| 你只想处理那些还没归属的文章 | `仅无归属` |

### 操作步骤

1. 点击左侧边栏底部的 **「↩ 匹配回填」** 按钮
2. 选择回填模式：
   - **全部标签**：重新计算所有标签的板块归属（最彻底，但可能比较慢）
   - **仅无归属**：只处理还没有归入任何板块的标签
   - **指定板块**：只重算可能属于某个特定板块的标签
3. 确认后任务开始执行，底部出现进度条
4. 进度条实时更新：`回填: 120/150  失败 0`
5. 完成后自动消失

### 进度说明

- 回填是后台异步执行的，你可以继续浏览其他内容
- 回填期间如果关闭页面，任务会继续在后台跑
- 重复触发回填不会出问题（幂等）

## 6. 辅助标签治理

### 打开位置

在右侧下方「辅助标签池」面板。

### 搜索和筛选

面板顶部有两个控件：
- **搜索框**：输入关键词模糊匹配小标签名（如搜"AI"）
- **状态下拉**：`全部状态` / `活跃` / `已禁用`

### 禁用小标签

当 LLM 生成了质量不好的小标签（比如「事件」「新闻」这种太泛的词），你可以禁用它：

1. 在列表中找到该小标签
2. 点击右侧的 **眼睛图标**（👁️‍🗨️ → 禁用）
3. 确认弹窗：「禁用后不再参与 board 匹配和升级候选」
4. 禁用的标签会变灰，不再被系统使用

> 被禁用的标签不会自动删除。如果想重新启用，筛选状态下拉选「已禁用」找到它再启用。

### 手动合并别名

有时候 LLM 生成了「AI」「人工智能」「Artificial Intelligence」三个标签，但本质是同一个东西。你可以手动合并：

1. 在列表中找到要合并的源标签，点击 **合并图标（⇄）**
2. 列表进入合并模式，每行出现「合并到此」按钮
3. 选择目标标签（被保留下来的那个），点击 **「合并到此」**
4. 确认弹窗：「将标签合并为 "XXX" 的 alias？」
5. 源标签的名字加入目标标签的别名列表，之前关联源标签的文章自动迁到目标

> 合并操作不可撤销，请确认后再操作。

### 聚类浏览

如果有聚类数据，面板顶部会显示聚类 tabs（如「全部」「簇1: AI相关」「簇2: 地缘政治」）。点击某个簇只看该簇内的小标签。这在标签量大的时候很有用。

## 7. 匹配参数配置

### 打开

点击左侧边栏底部的 **「⚙ 匹配参数」** 按钮。

### 参数说明

| 参数 | 默认值 | 通俗解释 |
|---|---|---|
| **相似度阈值** | 0.6 | 一个小标签跟板块"有多像"才算命中。调低 → 更容易挂载；调高 → 要求更严格 |
| **直接命中率阈值** | 0.5 | 标签有几个小标签命中了板块（比例），超过这个值直接挂载 |
| **直接匹配最大相似度** | 0.8 | 只要有一个小标签跟板块特别像（≥0.8），直接挂载 |
| **相似度权重** | 0.6 | 加权分中"最像"的占比 |
| **密度权重** | 0.4 | 加权分中"命中率"的占比 |
| **加权综合阈值** | 0.5 | 加权分达标线 |
| **最大归属板块数** | 3 | 一个标签最多归几个板块 |
| **升级引用数阈值** | 5 | 小标签被用了多少次才够格进入升级候选 |

### 怎么调

1. 打开配置弹窗
2. 修改参数（建议一次只改一两个，方便观察效果）
3. 点击 **「保存」** — 新参数对**之后处理的新标签**立即生效
4. 想让历史标签也按新规则重新算？→ 保存后手动触发一次 **「匹配回填」**

### 建议调参策略

- **标签太多归不进板块** → 降低「相似度阈值」（从 0.6 调到 0.5）
- **标签归到了不相关的板块** → 提高「相似度阈值」（从 0.6 调到 0.7）
- **板块太少迟迟不出升级建议** → 降低「升级引用数阈值」（从 5 调到 3）
- **升级建议太多太碎片化** → 提高「升级引用数阈值」（从 5 调到 8）

## 8. 浏览小标签池

### 面板位置

右侧下方的「辅助标签池」面板。这个面板展示系统从文章里提取到的所有"小标签"。

### 面板结构

```
┌─ 辅助标签池 ────────── 156 ──┐
│  [全部] [AI相关 35] [地缘政治 28] [金融 22] ...  ← 聚类tabs
│                               │
│  [🔍 搜索标签...]  [全部状态 ▾] │  ← 搜索+筛选
│                               │
│  AI                    12 引用  [⇄] [👁]  ← 每行一个标签
│  大语言模型            8 引用   [⇄] [👁]
│  OpenAI                6 引用   [⇄] [👁]
│  ...                          │
│                               │
│  ◀ 1  2  3  ...  8  ▶         │  ← 分页
└───────────────────────────────┘
```

### 每行信息

- **标签名**：如「AI」
- **aka 别名列表**：如果有别名会显示，如 `aka 人工智能, artificial intelligence`
- **引用次数**：被多少标签引用过（数字越大越热门）
- **操作按钮**：禁用（眼睛图标）、合并（合并图标）

### 使用场景

- **看看系统学到了什么**：浏览标签池，了解系统从你的订阅中提取了哪些概念
- **清理垃圾标签**：禁用太泛的标签（如「新闻」「事件」「报道」）
- **合并重复标签**：把同义词合并
- **按聚类浏览**：点击聚类 tab 只看某一类标签

## 常见问题

### Q: 为什么创建了板块但没有文章归入？

A: 有两种可能：

1. **还没触发回填**：创建板块后，历史文章不会自动归入。需要手动点击「匹配回填」
2. **构成标签太少或不合适**：板块的构成标签跟文章的小标签没有交集。尝试添加更多相关的小标签到板块构成中

### Q: 一个标签归了太多板块怎么办？

A: 在「匹配参数」中调低「最大归属板块数」（从 3 降到 2 或 1）

### Q: LLM 升级建议的质量不好怎么办？

A: 升级建议相当于 AI 给出的"草稿"，你可以：
- 不通过的建议直接跳过
- 创建后手动编辑板块名称和描述
- 在构成标签面板中增删小标签来调整板块覆盖范围

### Q: 禁用了一个小标签后能不能恢复？

A: 可以。在辅助标签池的筛选下拉中选择「已禁用」，找到它后重新启用。

### Q: 合并别名操作能撤销吗？

A: 不能。合并别名是不可逆操作（源标签的关联全部迁走了），请确认后再合并。

### Q: 冷启动阶段没有板块，每日简报会受影响吗？

A: 没有板块时，每日简报不会从板块派生。但文章标签和小标签的提取照常进行。等你创建了第一批板块并回填后，每日简报就会开始生成。

### Q: 回填要多久？

A: 取决于标签数量。100 个标签通常在几十秒内完成。回填是后台异步执行的，你可以继续使用其他功能。
